# 👋 Contribution guidelines

This document describes how to contribute to ExtenRe CLI.

## 📖 Resources to help you get started

- The [documentation](/docs) explains how to use ExtenRe CLI.
- [Issues](https://github.com/LuisCupul04/extenre-cli/issues) are where we track bugs and feature requests.

## 🙏 Submitting a feature request

Features can be requested by opening an issue using the
[Feature request issue template](https://github.com/LuisCupul04/extenre-cli/issues/new?assignees=&labels=feature&projects=&template=feature_request.yml&title=feat%3A+).

## 🐞 Submitting a bug report

If you encounter a bug, open an issue using the
[Bug report issue template](https://github.com/LuisCupul04/extenre-cli/issues/new?assignees=&labels=bug&projects=&template=bug_report.yml&title=bug%3A+).

## 📝 How to contribute

1. Before contributing, it is recommended to open an issue to discuss your change with the maintainers.
2. Development happens on the `dev` branch. Fork the repository and create your branch from `dev`.
3. Commit your changes.
4. Submit a pull request to the `dev` branch, referencing any issues it closes.
5. Your pull request will be reviewed. Once approved, it will be merged into `dev` and included in the next release.

❤️ Thank you for considering contributing to ExtenRe CLI!
